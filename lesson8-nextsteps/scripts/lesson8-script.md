"Hay algo importante que todos los programadores descubren tarde o temprano."

"No existe programador que escriba código perfecto a la primera."

# 🚨 No le tengas miedo a los mensajes de error

Cuando algo falla, Java suele mostrar mensajes como:

> **Nota visual:** Mostrar ejemplos visuales de los errores con código incorrecto adrede.

```text
cannot find symbol
```

o

```text
';' expected
```

o incluso:

```text
NullPointerException
```

"Al principio pueden parecer intimidantes con mensajes de errores incomprensibles.

Pero en realidad son pistas."

Piensa en ellos como si fueran un detective intentando decirte dónde ocurrió el problema.

# 🔍 Lee el error completo

Muchos principiantes hacen esto:

1. Ejecutan el programa.
2. Aparece un error.
3. Entran en pánico.
4. Empiezan a borrar código al azar.

"Evitemos caer en eso."

En lugar de eso:

✅ Lee el mensaje.

✅ Busca la línea indicada.

✅ Revisa lo que cambiaste recientemente.

"La mayoría de los errores tienen una solución más sencilla de lo que parece."

---

# 🧹 Mantén tu código ordenado

Comparemos estos dos ejemplos.

---

Código difícil de leer:

```java
int a=5;
int b=10;
System.out.println(a+b);
```

---

Código más claro:

```java
int edadAlumno = 5;
int edadProfesor = 10;

System.out.println(
        edadAlumno + edadProfesor);
```

---

"Ambos funcionan.

Pero el segundo es mucho más fácil de entender.

Tu código no solo debe funcionar.

También debe poder leerse desde la perspectiva del desarrollador.

Incluso por ti mismo dentro de una semana. Debes saber lo que estás haciendo en todo momento."
